# AI Capabilities and Limitations

## Overview

AI tools can help learners draft, summarize, explain, organize, and revise ideas. They can also produce output that sounds confident while still being incomplete, outdated, unsupported, or wrong.

This guide introduces a beginner-friendly way to think about AI use: neither hype nor fear, but calibrated trust.

::: info Key idea
Use AI when it can support the learning task. Verify AI when accuracy matters. Avoid or limit AI when privacy, fairness, policy, or high-stakes judgment is involved.
:::

## Audience

| Audience | How this guide can help |
| --- | --- |
| Students new to AI tools | Learn when AI can support learning and when extra checking is needed. |
| Faculty and instructors | Introduce realistic expectations for AI-supported academic work. |
| Workshop participants | Practice deciding whether a task is suitable for AI assistance. |
| Teaching support staff | Use shared language for explaining responsible AI practice. |

## Learning Goals

After using this guide, learners should be able to:

- Name common tasks where AI can be useful.
- Identify tasks where AI output is less reliable.
- Explain why fluent writing is not the same as verified knowledge.
- Decide when to use, verify, or avoid AI.
- Apply basic review habits before using AI-generated output.

## What AI Tools Are Good At

AI tools are often useful for first drafts, structure, language support, and low-stakes exploration. They work best when the user provides clear context and checks the result.

| Useful area | What AI can help with | Human review still needed |
| --- | --- | --- |
| Brainstorming | Generate possible topics, examples, questions, or outlines | Choose what fits the real goal |
| Summarizing | Create a first-pass summary of provided material | Compare with the source |
| Explaining | Restate concepts in simpler language | Check accuracy and missing context |
| Revising | Suggest clearer wording, tone, or organization | Preserve meaning and voice |
| Formatting | Turn notes into tables, checklists, or structured drafts | Check that nothing important was changed |
| Coding support | Suggest code, explain errors, or draft small examples | Run and test the code |

## What AI Tools Are Not Reliable At

AI tools are less reliable when a task requires exact facts, current information, hidden context, institutional rules, or human judgment.

| Risk area | Why it matters | Safer practice |
| --- | --- | --- |
| Citations and sources | AI may invent or misstate titles, authors, links, or publication details. | Verify every source in an approved database or primary source. |
| Recent or local facts | AI may not have the latest or most context-specific information. | Check current official sources. |
| Statistics and dates | Specific numbers can sound precise while being wrong. | Trace numbers back to reliable evidence. |
| Long or complex context | Important instructions can be missed or diluted. | Put key context up front and repeat critical constraints. |
| Sensitive information | AI use may create privacy or policy concerns. | Do not enter private data unless the tool and setting are approved. |
| High-stakes decisions | AI output may miss ethical, legal, academic, or personal consequences. | Keep human experts and institutional guidance in control. |

## Common Misunderstandings

| Misunderstanding | Better way to think about it |
| --- | --- |
| "If it sounds polished, it must be correct." | Fluent output can still include errors or unsupported claims. |
| "AI is basically a search engine." | AI generates responses; it does not automatically provide verified sources. |
| "A longer prompt always works better." | Clear goals and review criteria matter more than length. |
| "AI can replace reading the source." | AI can support reading, but learners still need to compare output with source material. |
| "One good answer proves the tool is reliable." | AI behavior can vary; important tasks need repeated checking. |
| "AI feedback is neutral." | AI may agree too easily, miss context, or reflect biased assumptions. |

## Calibrated Trust: When to Use, Verify, or Avoid AI

Calibrated trust means adjusting how much you rely on AI based on the task, evidence, and possible consequences.

| Decision | Good fit | Examples |
| --- | --- | --- |
| Use AI | The task is low-risk, draft-oriented, and easy to review. | Brainstorm discussion questions, reorganize notes, simplify a paragraph. |
| Use and verify | The output may be helpful, but accuracy matters. | Summarize a reading, explain a course concept, suggest sources, debug code. |
| Avoid or limit AI | The task involves private data, high-stakes judgment, or rules that require human authority. | Grading decisions, sensitive student information, official policy advice, final research claims. |

::: tip Review habit
Before trusting an AI response, ask: Does this task require exact facts, source evidence, current information, private context, or human judgment?
:::

## Beginner Checklist

Before using AI output, ask:

- [ ] Did I give the AI a clear goal?
- [ ] Did I provide the context it needs?
- [ ] Does the output fit the actual task, not just the requested format?
- [ ] Are key claims, citations, dates, statistics, or URLs verified?
- [ ] Did I compare summaries or explanations with the source material?
- [ ] Could the output be missing important context?
- [ ] Is any private, sensitive, or restricted information involved?
- [ ] Does course, classroom, or workshop guidance allow this use?
- [ ] Can I explain what I accepted, changed, or rejected?

## Classroom or Workshop Use

This guide can support short activities about realistic AI use.

| Activity | How to use it |
| --- | --- |
| Capability sorting | Give learners a list of tasks and ask whether each is a strong, medium, or risky AI use case. |
| Polished-but-wrong discussion | Show a fluent AI response with issues and ask learners what needs checking. |
| Prompt improvement | Start with a vague prompt and revise it with clearer goal, context, and criteria. |
| Source comparison | Compare an AI summary with the original reading and mark what is accurate, missing, or unsupported. |
| Trust decision practice | Ask learners to choose: use AI, use and verify, or avoid AI for different scenarios. |

## Next Steps

To adapt this guide for teaching:

1. Choose two or three AI use cases that fit the course or workshop.
2. Add clear examples of tasks where AI is helpful and tasks where caution is needed.
3. Connect the guide to course policy, privacy expectations, and approved tools.
4. Pair this page with an AI output evaluation activity.
5. Ask learners to reflect on when human judgment should lead.

This guide is a starting point for responsible AI practice. It should be adapted to the learning context before classroom or workshop use.
