# AI Evaluation Beginner Guide

## Overview

AI output evaluation is the habit of checking AI-generated work before trusting, submitting, sharing, or reusing it.

For beginners, evaluation does not need to be technical. A small set of clear questions can help learners decide whether an AI response is accurate enough, relevant enough, and appropriate for the task.

::: info Teaching focus
The goal is calibrated trust: trust AI more when the evidence is strong, and verify more when the task is important, uncertain, or high-risk.
:::

## Audience

| Audience | How this guide can help |
| --- | --- |
| Students new to AI tools | Build habits for checking AI-generated answers before using them. |
| Faculty and instructors | Introduce practical AI review routines in assignments or class activities. |
| Workshop participants | Practice judging AI outputs with shared criteria. |
| Teaching support staff | Use a common vocabulary for explaining responsible AI practice. |

## Learning Goals

After using this guide, learners should be able to:

- Explain why fluent AI output still needs review.
- Define what a good answer should include before judging an AI response.
- Check an AI output against a source, assignment, rubric, or set of expectations.
- Identify missing, unsupported, unclear, or inaccurate parts of an AI response.
- Decide whether to revise, verify, reject, or discuss the output.

## Why AI Output Evaluation Matters

AI tools can produce helpful drafts, summaries, explanations, and suggestions. They can also produce answers that sound polished while still being incomplete, unsupported, or wrong.

Evaluation helps learners move from "this sounds good" to "this meets the task and has been checked."

| Risk | Beginner-friendly review habit |
| --- | --- |
| Confident but incorrect claims | Verify facts, citations, dates, statistics, and examples. |
| Missing context | Compare the output with the assignment, source text, or class expectations. |
| Generic answers | Check whether the response fits the audience, course level, and purpose. |
| Unclear quality standard | Define what a good answer should include before reviewing. |
| Sensitive or inappropriate content | Remove private information and check the use context before sharing. |

## Four-Step AI Output Review Workflow

### 1. Name the task

Ask: What was the AI supposed to do?

Examples:

- Summarize a reading
- Explain a concept
- Suggest discussion questions
- Revise a paragraph
- Draft feedback
- Help with code

### 2. Define what a good answer should include

Before judging the output, list the most important expectations.

| Task type | Possible criteria |
| --- | --- |
| Summary | Accurate main ideas, no invented details, clear structure |
| Explanation | Beginner-friendly language, useful example, no misleading claims |
| Writing revision | Better clarity, appropriate tone, preserved meaning |
| Source-based answer | Faithful to the source, supported claims, verified citations |
| Coding help | Runs correctly, fits the problem, handles likely errors |

### 3. Check the output against evidence

Compare the AI output with the material or criteria that matter for the task.

Look for:

- Missing ideas
- Unsupported claims
- Incorrect details
- Overconfident wording
- Invented sources or examples
- Mismatch with the assignment or audience

### 4. Decide what to do next

Choose the next action instead of accepting the output automatically.

| Decision | When it fits |
| --- | --- |
| Revise | The output is useful but incomplete, unclear, or too general. |
| Verify | The output includes claims, citations, facts, data, or code behavior that must be checked. |
| Reject | The output misses the task, invents information, or creates risk. |
| Discuss | The output is useful for class reflection, comparison, or error analysis. |
| Use with attribution or reflection | The output meets expectations and follows course or workshop guidance. |

## Beginner Checklist

Before using an AI-generated output, ask:

- [ ] Does it answer the actual task?
- [ ] Does it match the audience and expected format?
- [ ] Are the main ideas accurate?
- [ ] Are important claims supported by a source or evidence?
- [ ] Are citations, quotes, dates, statistics, and URLs verified?
- [ ] Is anything missing, vague, or misleading?
- [ ] Does it follow course, classroom, or workshop guidance?
- [ ] Is any private or sensitive information included?
- [ ] Can I explain what I kept, changed, or rejected?

## Copy/Paste Practice Prompt

Use this prompt to practice reviewing an AI-generated output. Replace the bracketed text before using it.

```text
I want to review an AI-generated output before using it.

Task:
[What was the AI supposed to do?]

Audience or class context:
[Who is this for? What course, topic, or learning situation matters?]

What a good answer should include:
[List 3 to 5 expectations, such as accuracy, source use, tone, format, or key ideas.]

AI output to review:
[Paste the AI-generated output.]

Please help me identify:
1. What works well.
2. What may be inaccurate, unsupported, missing, unclear, or too general.
3. What should be checked against the source, assignment, rubric, or evidence.
4. Two specific revisions that would make the output more trustworthy.
```

::: warning Keep human judgment in the loop
An AI review of an AI output is only a support step. For important work, a person still needs to check the evidence, context, and final decision.
:::

## Classroom or Workshop Use

This guide can support short activities that help learners practice responsible AI use.

| Activity | How to use it |
| --- | --- |
| Output comparison | Give learners two AI responses and ask which one better meets the criteria. |
| Source-check practice | Ask learners to compare an AI summary with the original reading. |
| Rubric alignment | Have learners judge whether an AI response meets assignment expectations. |
| Error spotting | Use a flawed AI output to find missing ideas, weak evidence, or overconfident claims. |
| Revision reflection | Ask learners to explain what they changed and why before submitting or sharing. |

## Common Mistakes to Watch For

- Judging an answer by how polished it sounds.
- Asking for citations and assuming they are real.
- Reviewing the output without first defining success criteria.
- Using AI-generated text without checking the source material.
- Letting a second AI review replace human judgment.
- Ignoring privacy, course policy, or classroom expectations.
- Treating one good result as proof that the AI will always perform well.

## Next Steps

To turn this guide into a class or workshop activity:

1. Choose one low-stakes task, such as reviewing a short AI summary.
2. Define three to five quality criteria for that task.
3. Give learners time to compare the AI output with the source or assignment.
4. Ask learners to revise the output or explain why they would not use it.
5. Close with a short reflection on verification, responsibility, and appropriate AI use.

For classroom use, adapt the examples to the course policy, assignment goals, approved tools, and privacy expectations.
