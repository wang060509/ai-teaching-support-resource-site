# Workshop Module Map

## Overview

This module map connects beginner AI literacy guides into possible classroom or workshop sessions. It is a planning resource, not a confirmed official program.

The map is designed to help instructors, facilitators, and teaching support staff choose a small sequence of topics that build from basic AI understanding toward responsible practice.

::: info Planning note
Modules can be used independently or combined into a short workshop series. Each module should be adapted to the audience, course policy, approved tools, and available time.
:::

## Audience

| Audience | How this map can help |
| --- | --- |
| Faculty and instructors | Choose AI literacy topics for a class session or short unit. |
| Workshop facilitators | Plan a beginner-friendly sequence with activities and review points. |
| Teaching assistants | Prepare support materials, discussion prompts, and checklists. |
| Students or peer mentors | Understand how AI literacy topics connect across responsible use, evaluation, and workflow. |

## Learning Goals

After using this module map, planners should be able to:

- Sequence beginner AI literacy topics from basic concepts to practical review habits.
- Connect AI capabilities, limitations, prompting, evaluation, and reflection.
- Choose activities that support responsible AI practice.
- Decide which modules are core and which are optional or advanced.
- Identify what needs manual review before classroom or workshop use.

## Suggested Session Sequence

This sequence can be adapted for a single workshop or a short series.

| Session | Focus | Possible activity | Output |
| --- | --- | --- | --- |
| 1 | What AI can and cannot reliably do | Sort tasks into use, verify, or avoid categories | Capability-risk chart |
| 2 | How to review AI output | Compare an AI answer with a source, assignment, or rubric | Review checklist |
| 3 | How to ask better questions | Rewrite a vague prompt with task, context, and criteria | Improved prompt |
| 4 | Responsible AI practice | Discuss privacy, policy, citation, and reflection scenarios | Use-and-review plan |
| 5 | AI-assisted workflow | Map a small academic or teaching task into steps | Workflow outline |
| 6 | Optional coding workflow | Plan a small AI-assisted project and define checks | Project brief or test checklist |

## Module Details

### Module 1: AI Capabilities and Limitations

| Element | Notes |
| --- | --- |
| Learning goal | Help learners understand when AI can support work and when extra caution is needed. |
| Key ideas | AI can draft, summarize, explain, and organize; it can also be wrong, incomplete, or overconfident. |
| Activity idea | Ask learners to classify example tasks as low-risk, needs verification, or avoid/limit AI. |
| Review point | Polished output is not the same as verified output. |

### Module 2: AI Output Evaluation

| Element | Notes |
| --- | --- |
| Learning goal | Teach learners to check AI-generated output against criteria, evidence, and context. |
| Key ideas | Define what a good answer should include before judging the output. |
| Activity idea | Compare an AI-generated summary with the original reading or assignment instructions. |
| Review point | Evaluation makes AI use more visible, reviewable, and trustworthy. |

### Module 3: Prompt Basics

| Element | Notes |
| --- | --- |
| Learning goal | Help learners write clearer prompts for learning and review tasks. |
| Key ideas | Useful prompts name the task, context, audience, constraints, and expected output. |
| Activity idea | Rewrite a vague prompt into a more specific prompt with review criteria. |
| Review point | Better prompts help, but they do not remove the need to verify output. |

### Module 4: Responsible Use and Reflection

| Element | Notes |
| --- | --- |
| Learning goal | Connect AI use with privacy, course expectations, transparency, and human judgment. |
| Key ideas | Learners should know what they used AI for, what they changed, and what they checked. |
| Activity idea | Review short scenarios and decide what is appropriate, risky, or not allowed. |
| Review point | Responsible AI practice includes verification, privacy awareness, and reflection. |

### Module 5: AI-Assisted Workflow

| Element | Notes |
| --- | --- |
| Learning goal | Move from single prompts to a repeatable workflow: plan, draft, review, revise, and document. |
| Key ideas | AI can support parts of a process, but humans define the goal and decide what is ready to use. |
| Activity idea | Map a small task, such as creating a study guide or class activity, into AI-supported steps. |
| Review point | Each step should include a check for accuracy, audience fit, and privacy. |

### Module 6: Optional AI Coding or Codex-Style Workflow

| Element | Notes |
| --- | --- |
| Learning goal | Introduce AI-assisted project work for learners ready to plan, test, and revise a small artifact. |
| Key ideas | Start with a small goal, provide context, review changes, and test the result. |
| Activity idea | Draft a simple project brief or test checklist before asking for code or technical help. |
| Review point | This module should stay low-stakes and optional unless the audience has the needed setup and support. |

## How the Guides Connect

The guides can be used as a learning path:

| Start with | Then add | Why it helps |
| --- | --- | --- |
| AI capabilities and limitations | AI output evaluation | Learners first understand risks, then practice checking output. |
| AI output evaluation | Prompt basics | Learners use review criteria to ask clearer questions. |
| Prompt basics | Responsible use and reflection | Learners connect better prompts with privacy, policy, and accountability. |
| Responsible use | AI-assisted workflow | Learners apply responsible practice across a multi-step task. |
| AI-assisted workflow | Optional coding workflow | Learners extend the same review habits to small technical projects. |

## Classroom or Workshop Use

| Format | Suggested use |
| --- | --- |
| 15-minute class activity | Use one task-sorting or output-review activity. |
| 45-minute workshop | Combine capabilities, evaluation, and prompt basics. |
| 90-minute workshop | Add responsible use scenarios and a short workflow planning activity. |
| Multi-session series | Build from AI literacy to evaluation, prompting, workflow, and optional coding support. |

## Adaptation Notes

- Keep early modules tool-light so learners can focus on judgment, not setup.
- Add course policy, attribution expectations, and approved-tool guidance before classroom use.
- Use generic or public examples unless private data handling has been approved.
- Avoid making coding workflows a required beginner module unless learners have the needed support.
- Treat advanced topics such as plugins, automations, connectors, and complex coding assistants as optional background.
- Keep reflection visible: learners should explain what AI helped with, what they checked, and what they changed.

## Next Steps

To use this map for planning:

1. Choose the audience and time limit.
2. Select two or three modules that fit the learning goals.
3. Add examples that are generic, public, or approved for teaching use.
4. Decide what learners should produce: checklist, prompt, review notes, reflection, or workflow plan.
5. Review the materials for privacy, accuracy, policy alignment, and accessibility.

This page should remain flexible. It is a guide for planning possible teaching sessions, not a fixed curriculum.
